# Firebase Re-Integration Plan for Mental Mantra

**Current State:** Offline-first architecture (mock packages, local SharedPreferences)  
**Target State:** Hybrid cloud-sync architecture (Firebase + local fallback)  
**Timeline:** 2-4 weeks (phased rollout recommended)

---

## Executive Summary

This plan enables **gradual transition** from offline-only to Firebase-backed cloud sync. Key principles:

1. **Keep local-first foundation** — offline functionality remains intact
2. **Add cloud sync gradually** — start with authentication, expand to Firestore
3. **Graceful degradation** — if cloud fails, app still works offline
4. **User data safety** — no data loss during migration

### Phase Overview

| Phase | Duration | Focus | Risk |
|-------|----------|-------|------|
| **Phase 1** | 1 week | Firebase init + Auth | Low |
| **Phase 2** | 1 week | Firestore schema + rules | Medium |
| **Phase 3** | 1 week | Sync engine (write/read) | High |
| **Phase 4** | 1 week | Conflict resolution + testing | High |

## Phase 2: Firebase Authentication (Week 1-2)

### 2.1 Implement Firebase Auth Service

```dart
// lib/services/auth_service.dart

import 'package:firebase_auth/firebase_auth.dart';

class AuthService {
  final FirebaseAuth _auth = FirebaseAuth.instance;
  
  // Stream of authentication state
  Stream<User?> get authStateChanges => _auth.authStateChanges();
  
  // Current user
  User? get currentUser => _auth.currentUser;
  
  // Sign up with email/password
  Future<UserCredential> signUpWithEmail(String email, String password) async {
    try {
      return await _auth.createUserWithEmailAndPassword(
        email: email,
        password: password,
      );
    } on FirebaseAuthException catch (e) {
      throw _handleAuthError(e);
    }
  }
  
  // Sign in with email/password
  Future<UserCredential> signInWithEmail(String email, String password) async {
    try {
      return await _auth.signInWithEmailAndPassword(
        email: email,
        password: password,
      );
    } on FirebaseAuthException catch (e) {
      throw _handleAuthError(e);
    }
  }
  
  // Sign in with Google
  Future<UserCredential> signInWithGoogle() async {
    try {
      final GoogleSignInAccount? googleUser = await GoogleSignIn().signIn();
      final GoogleSignInAuthentication? googleAuth = 
          await googleUser?.authentication;
      
      final credential = GoogleAuthProvider.credential(
        accessToken: googleAuth?.accessToken,
        idToken: googleAuth?.idToken,
      );
      
      return await _auth.signInWithCredential(credential);
    } on FirebaseAuthException catch (e) {
      throw _handleAuthError(e);
    }
  }
  
  // Sign out
  Future<void> signOut() async {
    await _auth.signOut();
    await GoogleSignIn().signOut();
  }
  
  // Error handling
  String _handleAuthError(FirebaseAuthException e) {
    switch (e.code) {
      case 'user-not-found':
        return 'No user found with that email.';
      case 'wrong-password':
        return 'Wrong password provided.';
      case 'weak-password':
        return 'Password is too weak.';
      case 'email-already-in-use':
        return 'Email already in use.';
      default:
        return e.message ?? 'An error occurred.';
    }
  }
}
```

```javascript
// firestore.rules

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Deny all by default
    match /{document=**} {
      allow read, write: if false;
    }
```

### 3.3 Create Firestore Service

```dart
// lib/services/firestore_service.dart

class FirestoreService {
  final FirebaseFirestore _db = FirebaseFirestore.instance;
  
  // Get current user ID
  String? get userId => FirebaseAuth.instance.currentUser?.uid;
  
  // Save journal entry to cloud
  Future<void> saveJournalEntryToCloud(JournalEntry entry) async {
    if (userId == null) throw Exception('User not authenticated');
    
    await _db
        .collection('journal_entries')
        .doc(userId)
        .collection('entries')
        .doc(entry.id)
        .set({
          'content': entry.content,
          'createdAt': entry.createdAt.toIso8601String(),
          'updatedAt': entry.updatedAt?.toIso8601String(),
          'tags': entry.tags,
          'moodScore': entry.moodScore,
        });
  }
  
  // Stream journal entries from cloud
  Stream<List<JournalEntry>> getJournalEntriesStream() {
    if (userId == null) return Stream.value([]);
    
    return _db
        .collection('journal_entries')
        .doc(userId)
        .collection('entries')
        .orderBy('createdAt', descending: true)
        .snapshots()
        .map((snapshot) {
          return snapshot.docs
              .map((doc) => JournalEntry.fromFirestore(doc))
              .toList();
        });
  }
  
  // Delete journal entry from cloud
  Future<void> deleteJournalEntryFromCloud(String entryId) async {
    if (userId == null) throw Exception('User not authenticated');
    
    await _db
        .collection('journal_entries')
        .doc(userId)
        .collection('entries')
        .doc(entryId)
        .delete();
  }
}
```

## Phase 4: Hybrid Sync Engine (Week 3-4)

### 4.1 Implement Sync Strategy

The key principle: **Local first, cloud sync in background**

```dart
// lib/services/sync_service.dart

class SyncService {
  final LocalDataService _local = LocalDataService();
  final FirestoreService _cloud = FirestoreService();
  
  // Sync journal entry (write local, then cloud)
  Future<void> syncJournalEntry(JournalEntry entry) async {
    // 1. Always save to local storage FIRST (offline guarantee)
    await _local.saveJournalEntry(entry);
    
    // 2. Try to sync to cloud (if user is authenticated and online)
    if (FirebaseAuth.instance.currentUser != null) {
      try {
        await _cloud.saveJournalEntryToCloud(entry);
      } catch (e) {
        // Cloud sync failed—user can still access data offline
        print('Cloud sync failed: $e');
        // Queue for retry later
        await _queueForRetry('journal_entry_save', entry.toJson());
      }
    }
  }
  
  // Download and merge cloud data
  Future<void> syncFromCloud() async {
    if (FirebaseAuth.instance.currentUser == null) return;
    
    try {
      final cloudEntries = await _cloud.getJournalEntriesSnapshot();
      final localEntries = await _local.getJournalEntries();
      
      // Merge strategy: cloud entry wins if newer
      for (var cloudEntry in cloudEntries) {
        final localEntry = localEntries
            .firstWhereOrNull((e) => e.id == cloudEntry.id);
        
        if (localEntry == null) {
          // New entry from cloud
          await _local.saveJournalEntry(cloudEntry);
        } else if (cloudEntry.updatedAt.isAfter(localEntry.updatedAt ?? localEntry.createdAt)) {
          // Cloud entry is newer
          await _local.saveJournalEntry(cloudEntry);
        }
        // else: local entry is newer or same, keep local
      }
    } catch (e) {
      print('Cloud sync failed: $e');
    }
  }
  
  // Retry failed syncs
  Future<void> _queueForRetry(String action, Map<String, dynamic> data) async {
    final prefs = await SharedPreferences.getInstance();
    final queue = jsonDecode(
      prefs.getString('sync_queue') ?? '[]',
    ) as List;
    queue.add({
      'action': action,
      'data': data,
      'timestamp': DateTime.now().toIso8601String(),
    });
    await prefs.setString('sync_queue', jsonEncode(queue));
  }
}
```

### 4.2 Conflict Resolution Strategy

**For this app, use timestamp-based conflict resolution:**

```dart
// lib/models/sync_metadata.dart

class SyncMetadata {
  final DateTime localLastUpdated;
  final DateTime? cloudLastUpdated;
  final String localVersion;
  final String? cloudVersion;
  
  SyncMetadata({
    required this.localLastUpdated,
    this.cloudLastUpdated,
    required this.localVersion,
    this.cloudVersion,
  });
  
  /// Determine which version should be used
  /// Returns: 'local', 'cloud', or 'merge'
  String resolveConflict() {
    if (cloudLastUpdated == null) return 'local';
    if (localLastUpdated.isAfter(cloudLastUpdated!)) return 'local';
    return 'cloud';
  }
}
```

### 4.3 Background Sync Service

```dart
// lib/services/background_sync.dart

class BackgroundSyncService {
  static const Duration _syncInterval = Duration(minutes: 5);
  Timer? _syncTimer;
  
  void startPeriodicSync() {
    _syncTimer = Timer.periodic(_syncInterval, (_) async {
      await _performSync();
    });
  }
  
  void stopPeriodicSync() {
    _syncTimer?.cancel();
  }
  
  Future<void> _performSync() async {
    // 1. Retry failed syncs from queue
    await _retryFailedSyncs();
    
    // 2. Download new data from cloud
    await _syncService.syncFromCloud();
    
    // 3. Log sync event
    print('Sync completed at ${DateTime.now()}');
  }
  
  Future<void> _retryFailedSyncs() async {
    final prefs = await SharedPreferences.getInstance();
    final queueJson = prefs.getString('sync_queue');
    if (queueJson == null || queueJson.isEmpty) return;
    
    final queue = jsonDecode(queueJson) as List;
    for (var item in queue) {
      try {
        // Attempt to sync based on action type
        if (item['action'] == 'journal_entry_save') {
          final entry = JournalEntry.fromJson(item['data']);
          await _firestore.saveJournalEntryToCloud(entry);
          queue.remove(item);
        }
      } catch (e) {
        print('Retry failed: $e');
      }
    }
    
    // Update queue
    if (queue.isEmpty) {
      await prefs.remove('sync_queue');
    } else {
      await prefs.setString('sync_queue', jsonEncode(queue));
    }
  }
}
```

## Phase 6: Deployment Strategy

### 6.1 Gradual Rollout

1. **Closed Beta** (5-10 users)
   - Test Firebase auth + basic sync
   - Monitor for crashes and data loss
   - Collect feedback

2. **Open Beta** (100+ users)
   - Expand to more users
   - Monitor Firebase usage and costs
   - Optimize based on real-world usage

3. **Production** (100% of users)
   - Roll out to all users
   - Keep old offline version as fallback

### 6.2 Version Strategy

```
v1.0.0 (Current)  — Offline-first, no cloud
v1.1.0 (Beta)     — Firebase auth + optional sync
```

### 6.3 Monitoring & Analytics

After Firebase rollout, monitor:

- **Auth conversion rate:** % of users who sign up
- **Sync success rate:** % of writes successfully synced
